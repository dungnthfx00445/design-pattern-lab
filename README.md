# Lab 2 – Design Patterns (TypeScript)

## 📂 Danh sách Design Patterns

### 🔹 Creational Patterns
1. **Singleton**  
   → Đảm bảo một class chỉ có một instance duy nhất trong toàn bộ ứng dụng.

2. **Factory Method**  
   → Tạo object mà không cần phụ thuộc vào class cụ thể.

3. **Abstract Factory**  
   → Tạo ra một nhóm (family) các object liên quan và tương thích với nhau.

4. **Builder**  
   → Xây dựng object phức tạp từng bước, tránh constructor có quá nhiều tham số.

5. **Prototype**  
   → Tạo object mới bằng cách clone object có sẵn.

---

### 🔹 Structural Patterns
6. **Adapter**  
   → Chuyển đổi interface không tương thích để client có thể sử dụng.

7. **Decorator**  
   → Thêm hành vi cho object một cách linh hoạt mà không thay đổi interface.

---

### 🔹 Behavioral Patterns
8. **Observer**  
   → Thiết lập mối quan hệ một–nhiều giữa các object, thường dùng trong event / listener.

---
